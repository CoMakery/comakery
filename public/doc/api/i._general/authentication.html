AUTHENTICATION

GET /api/v1/projects

Requests should include `API-Public-Key` header and a correct proof based on `Ed25519Signature2018` in the format described below. All values should be strings. Note: When calculating the signature, request data should be serialized according JSON Canonicalization Scheme.

Parameters

Name Description
body[data] request data
body[url] request url
body[method] request http method
body[nonce] request nonce (rotated every 24h)
body[timestamp] request timestamp (expires in 60 seconds)
proof[type] Ed25519Signature2018
proof[verificationMethod] public key
proof[signature] request signature
